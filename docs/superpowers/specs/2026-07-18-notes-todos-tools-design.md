# 笔记与 Todo 工具设计

## 背景

CDY Agent 已交付双 API 模式的 Agent Tool Loop，以及受 workspace 限制的文件和 Shell 工具。第五阶段将在现有工具注册、参数预检、用户确认和本地安全边界之上，加入面向个人助理的笔记与 Todo 能力。

笔记与 Todo 需要在进程退出后保留，但本阶段不提前实现会话持久化、长期记忆或通用数据库层。数据因此保存到当前 workspace 的专用目录，并使用简单、可检查的版本化 JSON 格式。

## 目标与非目标

### 目标

- 提供创建、列表、查看和删除笔记的模型工具。
- 提供创建、列表、完成和删除 Todo 的模型工具。
- 将数据持久化到当前 workspace 下的 `.cdy-agent` 目录。
- 所有创建、完成和删除操作逐次请求用户确认，默认拒绝。
- 列表和查看操作自动执行。
- 对参数、存储结构、路径和符号链接实施明确限制。
- 使用原子文件替换降低写入中断导致数据损坏的风险。
- 使用完全离线的测试覆盖正常路径、安全边界和失败行为。

### 非目标

- 不支持编辑笔记、编辑 Todo、搜索、标签、优先级或截止日期。
- 不支持多个进程并发写入同一 workspace。
- 不实现会话持久化、长期记忆、SQLite 或数据库迁移框架。
- 不允许模型或用户为这些工具指定任意存储路径。
- 不自动修复或覆盖损坏、未知版本或结构异常的数据文件。
- 不新增 CLI 子命令；笔记与 Todo 通过现有 Agent Tool Loop 使用。

## 架构与职责

新增模块保持现有工具边界：

- `tools/personal_store.py` 管理专用数据目录、版本化 JSON 文档、数据校验和原子写入。
- `tools/notes.py` 实现笔记参数预检、确认说明和操作语义。
- `tools/todos.py` 实现 Todo 参数预检、确认说明和操作语义。
- `tools/__init__.py` 将新工具按确定顺序加入内置 Registry。
- `.gitignore` 忽略根目录及任意子 workspace 中的 `.cdy-agent/` 运行期数据目录。

CLI、Agent 和 OpenAI 客户端不增加笔记或 Todo 专用逻辑。Registry 继续负责 JSON 参数解码、预检、确认和执行；工具结果继续使用现有 `ToolResult` 结构回传模型。

`PersonalStore` 只服务本阶段的笔记与 Todo，不演化为会话、记忆或任意实体的通用持久化抽象。

## 工具接口

内置 Registry 新增以下八个函数工具，顺序固定为笔记工具后 Todo 工具。

### `create_note`

参数：

- `title: str`：去除首尾空白后必须非空，最多 200 个字符。
- `content: str`：允许为空，最多 64 KiB UTF-8 字节。

行为：创建包含稳定 UUID、标题、正文和 UTC 创建时间的笔记。执行前展示标题和正文 UTF-8 字节数并请求确认。

### `list_notes`

无参数。返回全部笔记的 `id`、`title` 和 `created_at`，不返回正文。结果按创建时间升序排列；相同时间以 ID 升序作为稳定次序。

### `get_note`

参数：

- `note_id: str`：规范 UUID 字符串。

行为：返回指定笔记的全部字段，无需确认。

### `delete_note`

参数：

- `note_id: str`：规范 UUID 字符串。

行为：记录必须在预检时存在；执行前展示 ID 和标题并请求确认。确认后重新读取最新数据，仅删除同一 ID 的记录。

### `create_todo`

参数：

- `text: str`：去除首尾空白后必须非空，最多 1,000 个字符。

行为：创建包含稳定 UUID、文本、未完成状态和 UTC 创建时间的 Todo。`completed_at` 初始为 `null`。执行前展示 Todo 文本并请求确认。

### `list_todos`

无参数。返回全部 Todo 字段，按创建时间升序排列；相同时间以 ID 升序作为稳定次序。首版不提供状态过滤参数。

### `complete_todo`

参数：

- `todo_id: str`：规范 UUID 字符串。

行为：记录必须在预检时存在且尚未完成；执行前展示 ID 和文本并请求确认。确认后重新读取最新数据，将 `completed` 设为 `true` 并写入 UTC `completed_at`。若记录在确认期间已被完成，则返回 `todo_already_completed`。

### `delete_todo`

参数：

- `todo_id: str`：规范 UUID 字符串。

行为：记录必须在预检时存在；执行前展示 ID 和文本并请求确认。确认后重新读取最新数据，仅删除同一 ID 的记录。

所有工具 schema 都使用 `additionalProperties: false`。无参数工具只接受空对象。字符串长度按上述规则校验，正文上限按 UTF-8 字节数计算。工具不接受隐式类型转换。

## 数据格式与排序

专用目录和文件为：

```text
<workspace>/.cdy-agent/
├── notes.json
└── todos.json
```

两个文件均为版本化 JSON 文档：

```json
{
  "version": 1,
  "items": []
}
```

笔记记录字段固定为：

- `id`: 规范 UUID 字符串。
- `title`: 非空字符串。
- `content`: 字符串。
- `created_at`: 带 `Z` 后缀的 UTC ISO 8601 时间。

Todo 记录字段固定为：

- `id`: 规范 UUID 字符串。
- `text`: 非空字符串。
- `completed`: 布尔值。
- `created_at`: 带 `Z` 后缀的 UTC ISO 8601 时间。
- `completed_at`: 完成时为 UTC ISO 8601 时间，否则为 `null`。

读取时严格校验顶层对象、版本、字段集合、字段类型、UUID、时间和 Todo 状态一致性。未知字段、重复 ID、`completed` 与 `completed_at` 不一致或任何非法记录都会使整个操作失败，不返回部分数据。

列表操作在返回前显式排序，不依赖文件中的物理顺序。创建时间相同时使用 ID 作为第二排序键。

## 路径与写入安全

workspace 在 CLI 启动时已经解析为真实目录。`PersonalStore` 在每次访问时验证 `.cdy-agent` 及目标数据文件的真实位置：

- 如果 `.cdy-agent` 不存在，读取视为空集合且不创建任何内容。
- 首次已确认的写操作可在 workspace 内创建 `.cdy-agent`。
- 如果 `.cdy-agent` 已存在但不是普通目录，操作失败。
- 如果 `.cdy-agent` 或数据文件通过符号链接解析到 workspace 外，操作返回 `path_outside_workspace`。
- 如果数据文件存在但不是普通文件，操作失败。
- 工具接口不暴露路径参数。

写入在 `.cdy-agent` 内创建同文件系统的临时文件，完整写入 UTF-8 JSON、刷新并成功关闭后使用 `os.replace` 原子替换目标文件。写入或替换失败时返回结构化存储错误，现有目标文件保持不变；可清理的临时文件由失败路径清理。

本阶段不提供跨进程锁。文档明确同一 workspace 同时只能由一个 `cdy-agent` 进程修改，避免以不完整的锁策略制造错误安全感。

## 确认与数据流

一次有副作用的工具调用遵循：

1. Registry 解码 JSON 参数。
2. 工具预检参数、存储状态和目标记录。
3. 无效调用直接返回结构化错误，不显示确认提示。
4. Registry 使用工具生成的具体说明请求用户确认。
5. 用户拒绝时返回 `approval_denied`，不创建目录或文件。
6. 用户同意后，工具重新读取和校验最新数据。
7. 工具应用单个变更并原子写入对应 JSON 文件。
8. 结构化结果通过现有 Agent Loop 回传模型。

重新读取不能解决多进程写入竞争，但可以避免在用户思考确认期间无条件使用过期的内存快照。目标记录在确认后消失时返回对应的 `*_not_found`；Todo 在确认后已完成时返回 `todo_already_completed`。

列表、查看和失败的预检没有副作用，不请求确认。所有确认继续使用 CLI 的默认 No 行为，EOF、键盘中断和不明确输入均视为拒绝。

## 错误处理

工具使用简短稳定的错误码，包括：

- `invalid_arguments`：参数集合、类型、长度或 UUID 不合法。
- `note_not_found`：指定笔记不存在。
- `todo_not_found`：指定 Todo 不存在。
- `todo_already_completed`：指定 Todo 已完成。
- `path_outside_workspace`：专用目录或数据文件越过 workspace。
- `invalid_store`：JSON、版本、编码或存储结构非法。
- `store_error`：目录、读取、临时写入或原子替换等 I/O 操作失败。
- `approval_denied`：用户拒绝副作用操作；由现有 Registry 产生。

错误消息可供模型解释，但不包含数据文件正文、环境变量、API 凭据或无关内部异常细节。损坏文件绝不被当作空集合，也不会被后续写操作覆盖。

## 测试策略

测试只使用 pytest 临时目录、固定时钟或可注入 ID/时间生成器，以及模拟的失败边界；不读取用户文件、不使用真实 API 或网络。

### Store 测试

- 空 workspace 读取为空且不创建 `.cdy-agent`。
- 首次写入创建专用目录和正确的版本化 JSON。
- 新 Store 实例可读到已保存数据。
- 符号链接目录和数据文件不能逃离 workspace。
- 损坏 JSON、非 UTF-8、未知版本、错误字段、重复 ID 和状态不一致均失败。
- 原子替换失败时保留原始文件并清理临时文件。

### 工具测试

- 八个工具的名称、schema、确认要求和注册顺序。
- 笔记创建、列表摘要、查看和删除。
- Todo 创建、列表、完成和删除。
- 稳定 ID、UTC 时间、排序及完成时间。
- 空白文本、字段长度、正文 UTF-8 字节上限、多余字段和错误类型。
- 不存在记录、重复完成，以及确认前后的状态变化。
- 用户拒绝时不创建或修改存储。

### 集成与回归测试

- Agent 能调用新工具、回传结果并生成最终回复。
- CLI 对所有副作用展示具体说明，默认拒绝路径无副作用。
- README 说明数据目录、操作范围、确认规则和单进程限制。
- `.gitignore` 防止 `.cdy-agent/` 中的个人数据被误提交。
- 完整离线测试、CLI help 和构建继续通过。

## 验收标准

- 模型可通过现有两种 API 模式使用八个笔记与 Todo 工具。
- 数据跨 `cdy-agent` 进程保留在当前 workspace 的 `.cdy-agent` 目录。
- 所有创建、完成和删除操作必须在执行前获得逐次确认。
- 列表和查看不产生存储副作用。
- 专用存储不能通过路径或符号链接逃离 workspace。
- Git 默认忽略 workspace 中的 `.cdy-agent/` 个人数据目录。
- 无效参数和损坏存储产生稳定错误，且不会破坏或覆盖已有数据。
- 自动测试完全离线，完整 pytest、CLI help 和构建全部通过。
